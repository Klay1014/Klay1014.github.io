這篇紀錄我最近用 LSTM 預測 0050（元大台灣卓越 50 ETF）收盤價的嘗試過程，主要目的是看看能不能靠技術指標輔助、讓 LSTM 模型學出合理的趨勢。過程中試了不同的預測目標和資料處理方法，最終還是決定回歸原始的 close 價。

資料來源與前處理#

資料使用 yfinance 抓取 2005～2025 的 0050 日線資料：

df = yf.download("0050.TW", start="2005-01-01", end="2025-06-06")

先確認資料大致形式，畫出時間變化的折線圖

接著加上幾個常見技術指標：

  • OBV (On-Balance Volume)：用成交量追蹤價格趨勢
  • MFI (Money Flow Index)：結合成交量與價格變動的震盪指標
  • Volume SMA (5 日均線)：平滑成交量波動

這些欄位都轉成 float 型別，並移除空值。

features = ['Open', 'Close', 'OBV', 'MFI', 'Volume', 'Volume_SMA_5']
df = df[features].dropna()

特徵正規化與序列建立#

採用 MinMaxScaler 對所有特徵進行 0～1 正規化。之後將資料轉為以過去 30 天資料預測未來 1 天 close 價的序列格式：

SEQ_LEN = 30
X, y = create_sequences(scaled, SEQ_LEN)

訓練測試集比例為 80:20，確保模型能在近年資料做出推論。

LSTM 模型架構#

模型結構維持簡單：

  • LSTM(64)
  • Dropout(0.2)
  • Dense(32 relu)
  • Dense(1)
model = Sequential([
    LSTM(64, input_shape=(30, 6), return_sequences=False),
    Dropout(0.2),
    Dense(32, activation='relu'),
    Dense(1)
])

訓練 50 epochs，batch size 為 32。

評估指標#

📊 Evaluation Metrics:
✅ MSE   = 117.2363
✅ MAE   = 9.8407
✅ R²    = 0.8894

MSE 和 MAE 表現還可以接受，R² 達到 0.88 算是不錯，代表模型能解釋大部分變異。

總結與想法#

曾經試過用報酬率 (return) 作為預測目標，但發現反而會讓模型難以收斂或爆炸，預測圖非常不合理。因此還是回到直接預測 close 價，表現最穩定。

接下來如果有時間會考慮：

  • 多加一層 LSTM
  • 增加更多技術指標（例如 MACD 或 RSI）
  • 嘗試 Transformer 類結構
