Excel 打卡資料分析實作心得：從 Real data 整理出 Insights

有鑒於 Excel 是很常見的分析工具，也不難上手，這次就嘗試做一個有感的實作：從 Kaggle https://www.kaggle.com/datasets/yasirub/employee-attendance-ratings?resource=download 下載一份包含 731 個日期、100 位員工的全年打卡資料（數量達 73,100 筆），拿來練習常見功能，例如 VBA 展開、長表轉換與樞紐分析。 一開始，第一眼就被那張 731 x 101 的巨型試算表震懾了。這些數據裡，每一格都記錄著一個人的上下班時間： 有人早上 08:43 就打卡、也有人總是 09:18 才出現；有些則忘記沒打到卡。 於是我做的第一件事是把它轉成「可分析的長表格式」，也就是： 每一列都是「日期 + 員工代碼 + 上班時間 + 狀態」 狀態會標記為「準時」、「遲到」、「缺卡」 這件事我用 VBA 一次跑完，轉出總筆數是 71,200，比預期少了 1,900 筆， 後來才發現那是 12 月底幾天的資料原本就空白，讓我更確信「資料檢查永遠該是第一步」。 接下來，我從這些資料中試著拉出一些簡單的指標： ✅ 員工遲到排行榜 使用樞紐分析計算每個人「遲到」的總次數，再進一步做 Top 10 排名。 有些人遲到次數超過 700，幾乎天天遲到。這個榜單的意義其實不光是針對員工本身，對於管理者來說，也是一種績效與風氣的觀察依據。 ✅ 每月遲到人數趨勢圖 我把狀態限定為「遲到」，然後用「月份」為欄位，拉出一張趨勢圖。 可以看出整體上班風氣算穩定，但在 2024 年 12 月有明顯下降，調查之後才發現該月資料只到 12/12，數據不完整。 這也是視覺化的一個意義——它會逼你去發現異常，從而回到資料源頭重新檢查。 ✅ 平均上班時間排行 將打卡時間轉換成分鐘（例如 08:45 就是 525），使用 HOUR()*60 + MINUTE() 實作，然後再用「平均值」來比較員工們的上班習慣。 ...

從 Hugo 到 GitHub Pages：部落格從零開始的記錄

老實說，會開始架這個部落格，其實只是因為最近的腦袋有點亂，覺得是時候需要一個地方，好好整理一下自己在學習、工作與生活中不斷累積的東西。 社群媒體太吵、筆記工具太私人，於是乾脆來開一個自己的網站好了。 因此我找到了 Hugo，一個號稱「最快的靜態網站產生器」。一開始其實完全看不懂它的結構，也不知道該從哪下手，但 Google 幾次、撞牆幾次後，大概摸到點門路。 從終端機開始，到網頁出現 跑了 hugo new site myblog，網站就出現了一個骨架。 我以為這樣就可以開網頁了，結果當然不可能。還得加主題，還得設定 config，還得理解什麼是 public/、什麼是 draft。 整個流程有點像是：你拿到一張白紙、一支筆、幾行範例，然後被丟到森林裡，要你自己蓋房子。雖然一開始有點懵，但其實也挺好玩的。 後來我選了 PaperMod 主題，因為夠簡單、夠乾淨，還可以有個自介和頭像，不會像預設樣板那麼冰冷。 發佈，比想像中更複雜一點 Hugo build 出來的內容會被放到 public/ 裡，而 GitHub Pages 要的就是這個資料夾。 但這代表你不能只把整個 Hugo 專案推上去，而是要另外把 public/ 當成一個 git repo，推到像 Klay1014.github.io 這種 repo 裡。這邊卡了不少時間。 一開始還會遇到一些莫名其妙的 404、CSS 沒載到、主題沒套上，各種「看起來沒錯但就是壞掉」的狀況。 解法就是慢慢試，慢慢 Google，然後養成一種「出錯也沒關係，總會修好」的肌肉記憶。 到這裡為止，我有了這些 成功讓網站跑起來（甚至有了自己的首頁、自我介紹、導覽列） 學會怎麼寫 Markdown 當文章、怎麼分類、怎麼標籤 每次修改後，跑 ./deploy.sh 就能一鍵發佈 這一切雖然很技術，但某種程度上，也很像在整理房間。 把一堆零散的筆記、想法、碎碎念，變成一個可以讓自己安頓的空間。 如果你剛好也想寫東西 歡迎你一起試試 Hugo。雖然不是最簡單的工具，但它乾淨、快速，而且夠彈性。 部落格不是什麼了不起的東西，但如果它能幫助我偶爾停下來，把腦子裡的東西倒一倒，那也很夠了。
