從零開始建立我的 Hugo 部落格

其實我一開始根本不知道 Hugo 是什麼。 就是某天突然覺得：「嗯……想弄個部落格來記錄一下東西。」 然後查了一下，發現 Hugo 是個很熱門的靜態網站產生器，而且搭配 GitHub Pages 就能免費上線。 接著就開始踩雷之旅 🤯 建立專案 我照官方文件跑 hugo new site myblog，雖然看起來都 OK，但一開始連要怎麼讓首頁長得好看都不知道。後來才知道要裝主題，我選了 ananke，還以為裝好就會變有內容，結果空的 🤡 後來換成了 PaperMod，整個視覺和結構清楚很多，也支援暗黑模式，還可以自訂首頁介紹，看起來比較「部落格」。 發佈到 GitHub Pages 這邊也卡了一下。一開始我傻傻地把整個專案推上 GitHub，然後發現網頁沒東西。才知道原來要把 public/ 裡面 Hugo build 出來的結果單獨推到 Klay1014.github.io 這個 repo 才會顯示。 GitHub Pages 的設定也得手動設 branch / folder，不設的話會一直看到 404。 到現在為止 目前首頁、導覽列、About 頁面都設好了，也寫了這篇文章。還學到怎麼用 menu 加上導覽列，怎麼關掉 draft，怎麼讓 Markdown 檔出現在網頁上。 整體來說 Hugo 蠻快的，但需要花一點時間摸熟配置邏輯。不過，當我第一次看到自己網站成功上線那一刻，真的超爽！ 之後可能會繼續加上文章分類、Google Analytics，甚至研究一下留言功能。不知道能不能堅持寫下去，但有個地方可以記錄，是個不錯的開始。
